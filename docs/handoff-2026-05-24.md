## Arquitetura resumida

```
GitHub (push main)
    ↓ webhook (a configurar)
n8n na VPS
    ↓ Execute
nexus-deploy.sh (/usr/local/bin)
    ↓ git pull + docker build + docker up + apply migrations
nexus-web container (Next.js 15)
    ├── DB: nexus-db (Postgres 16)
    └── Behind: Traefik (compartilhado com OpenClaw, network pg-cloud-net)
```

### Componentes

- **Workspaces (Aplicativos)**: agrupador lógico (Vision, Mapa Comparativo, etc)
- **Connections**: 1 Workspace → N Connections; cada Connection = 1 provider config
- **Resources**: descobertos automaticamente por Connection (Firebase project, hosting sites, functions, repos GitHub, etc)
- **Tenants**: dentro de Resources que são multi-tenant (Firebase Auth tenants)
- **Clients**: globais; alocados a Resources/Tenants p/ rateio
- **Incidents**: gerados automaticamente por health check, cost spike, idle, collection_failed
- **CostSnapshot**: serie diária por (Resource, Date, Source) com breakdown opcional por serviço

### Providers implementados

| Provider | Recursos | Health | Custo | Tests |
|----------|----------|--------|-------|-------|
| `firebase` | project + hosting + functions | URL probe + state | Cloud Monitoring (free) + BigQuery export | ✓ |
| `supabase` | projects | health endpoint | Management API billing | ✓ |
| `vercel` | projects | productionUrl probe | ❌ (API não expõe) | ✓ |
| `github` | repos | sempre OK | ❌ | ✓ |
| `cloudflare` | zones + workers | zone hostname probe | ❌ | ✓ |
| `azure` | subscriptions / app services | defaultHostName probe | ❌ (Cost Mgmt API TODO) | ✓ |
| `docker` | containers VPS | state | ❌ | ✓ |

## Como fazer deploy

### Opção A — Auto-deploy (após setup webhook n8n)

```bash
git push origin main
```

Pronto. n8n recebe webhook, roda `nexus-deploy.sh` na VPS, build + restart + migrations automáticas. Log em `/var/log/nexus-deploy.log`.

### Opção B — Manual (até webhook estar funcionando)

SSH na VPS + cola:

```bash
cd /docker/nexus && git pull && docker compose build nexus-web && docker compose up -d nexus-web && sleep 12 && docker compose ps
```

Se migration nova:

```bash
cat prisma/migrations/<latest>/migration.sql | docker compose exec -T nexus-db psql -U nexus -d nexus && docker compose restart nexus-web
```

## Recovery scenarios

### Perdeu NEXUS_MASTER_KEY
- Service account Firebase cifrado no banco vira ilegível
- Solução: deletar Connection, recriar via wizard (re-upload JSON)
- Snapshots de custo + inventário permanecem (não dependem da master key)

### VPS quebra
- Snapshot Hostinger no hPanel (1 incluso no plano) → restore
- OU clean install + clone repo + restaurar `.env` do cofre

### Senha root VPS perdida
- hPanel → Settings → Root password → Change → atualizar `hostinger-senha-root.txt`
- Não afeta SSH key auth se time usa keys

### Banco corrompido
- `pg_dump` cron deveria rodar (TODO: confirmar configuração)
- Se sem backup: snapshot Hostinger é fallback
